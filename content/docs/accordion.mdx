# Accordion

Accordion organizes related sections where each trigger expands its matching
panel. Behavior and accessibility come from Base UI; the public API matches
shadcn's four-part anatomy: `Accordion`, `AccordionItem`, `AccordionTrigger`,
and `AccordionContent`.

<Box display="flex" flexDirection="column" gap="m" padding="l" backgroundColor="background-card" borderRadius="m" border borderColor="border-primary">

  <Accordion defaultValue={["item-1"]}>
    <AccordionItem value="item-1">
      <AccordionTrigger>Is it accessible?</AccordionTrigger>
      <AccordionContent>
        Yes. Base UI wires the trigger, panel, keyboard behavior, and ARIA state.
      </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-2">
      <AccordionTrigger>Can it be styled?</AccordionTrigger>
      <AccordionContent>
        Yes. Use tokens, owned source edits, or the typed `sx` prop.
      </AccordionContent>
    </AccordionItem>
  </Accordion>

</Box>

## Usage

```tsx
import {
  Accordion,
  AccordionContent,
  AccordionItem,
  AccordionTrigger,
} from "@/components/accordion"
```

```tsx
<Accordion defaultValue={["item-1"]}>
  <AccordionItem value="item-1">
    <AccordionTrigger>Is it accessible?</AccordionTrigger>
    <AccordionContent>
      Yes. Base UI provides the behavior and accessibility.
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

## Customization

Accordion does not accept `className` or raw `style`. Use Base UI state props,
owned source edits, token themes, and the typed `sx` prop for on-system
customization.

```tsx
import * as stylex from "@stylexjs/stylex"
import { colors } from "@/styles/tokens.stylex"

const styles = stylex.create({
  muted: { backgroundColor: colors["background-muted"] },
})

<Accordion defaultValue={["one"]} sx={styles.muted}>
  <AccordionItem value="one">
    <AccordionTrigger>One</AccordionTrigger>
    <AccordionContent>Panel content.</AccordionContent>
  </AccordionItem>
</Accordion>
```
