# Badge

A compact status label styled with StyleX design tokens. It matches shadcn's
Badge variants while keeping customization on-system through `variant`, `sx`,
and token re-theming.

<Box display="flex" alignItems="center" gap="m" flexWrap="wrap" padding="l" backgroundColor="background-card" borderRadius="m" border borderColor="border-primary">
  <Badge>Default</Badge>
  <Badge variant="secondary">Secondary</Badge>
  <Badge variant="destructive">Destructive</Badge>
  <Badge variant="outline">Outline</Badge>
  <Badge variant="ghost">Ghost</Badge>
  <Badge variant="link" render={<Link to="/docs/$" params={{ _splat: "badge" }} />}>Link</Badge>
</Box>

## Usage

```tsx
import { Badge } from "@/components/badge"
```

```tsx
<Badge variant="outline">Badge</Badge>
```

## Customization

`Badge` does not accept `className` or `style`. Use `variant` first, then `sx`
for typed per-instance StyleX overrides, and `createTheme` to re-skin tokens for
a subtree.

```tsx
import * as stylex from "@stylexjs/stylex"

const styles = stylex.create({
  uppercase: { textTransform: "uppercase" },
})

<Badge sx={styles.uppercase}>Preview</Badge>
```

## API Reference

| Prop      | Type                                                                         | Default     |
| --------- | ---------------------------------------------------------------------------- | ----------- |
| `variant` | `"default" \| "secondary" \| "destructive" \| "outline" \| "ghost" \| "link"` | `"default"` |
| `sx`      | `StyleXStyles`                                                               | —           |

All other Base UI `useRender` span props are forwarded, including `render`.
