# Collapsible

A disclosure component that toggles a panel open and closed. Behavior and
accessibility come from Base UI's `Collapsible` primitive; shadcn parity is the
same three-part API: `Collapsible`, `CollapsibleTrigger`, and
`CollapsibleContent`. No `className`.

<Box display="flex" flexDirection="column" gap="m" padding="l" backgroundColor="background-card" borderRadius="m" border borderColor="border-primary">

  <Collapsible defaultOpen>
    <CollapsibleTrigger render={<Button aria-label="Toggle details" variant="outline" />}>
      Toggle details
    </CollapsibleTrigger>
    <CollapsibleContent>
      <Box paddingY="m" color="text-secondary">
        Collapsible content stays mounted only while open by default.
      </Box>
    </CollapsibleContent>
  </Collapsible>

</Box>

## Usage

```tsx
import {
  Collapsible,
  CollapsibleContent,
  CollapsibleTrigger,
} from "@/components/collapsible"
```

```tsx
<Collapsible>
  <CollapsibleTrigger>Toggle</CollapsibleTrigger>
  <CollapsibleContent>Content</CollapsibleContent>
</Collapsible>
```

## Examples

### With a Button trigger

Use Base UI's `render` prop to render the trigger as another shadcn-x primitive.
Do not pass `className`.

  <Collapsible>
    <CollapsibleTrigger render={<Button aria-label="Show release notes" variant="outline" />}>
      Show release notes
    </CollapsibleTrigger>
    <CollapsibleContent>
      <Box paddingY="m" color="text-secondary">
        Fixed keyboard behavior and improved panel measurement.
      </Box>
    </CollapsibleContent>
  </Collapsible>

</Box>

```tsx
<Collapsible>
```

### Default open

Set `defaultOpen` when the panel should render expanded initially.

    <CollapsibleTrigger render={<Button aria-label="Account settings" variant="outline" />}>
      Account settings
    </CollapsibleTrigger>
    <CollapsibleContent>
      <Box paddingY="m" color="text-secondary">
        Email notifications, billing details, and team access.

</Box>


### Controlled

Use `open` and `onOpenChange` when the parent owns the state.

```tsx
const [open, setOpen] = useState(false)

<Collapsible open={open} onOpenChange={setOpen}>
  <CollapsibleTrigger>Toggle</CollapsibleTrigger>
  <CollapsibleContent>Content</CollapsibleContent>
</Collapsible>
```

## Customization

`Collapsible` accepts no `className`. Customize it on-system:

1. **`sx`** — a typed, per-instance override on the root, trigger, or content.
2. **`render`** — style the trigger by rendering a component such as `Button`.
3. **`createTheme`** — re-skin a subtree by overriding token values.

```tsx
import * as stylex from "@stylexjs/stylex"

const styles = stylex.create({
  content: { maxWidth: "24rem" },
})

<Collapsible>
  <CollapsibleTrigger render={<Button aria-label="Toggle" variant="outline" />}>
    Toggle
  </CollapsibleTrigger>
  <CollapsibleContent sx={styles.content}>Content</CollapsibleContent>
</Collapsible>
```

## API Reference

### Collapsible

| Prop             | Type                                      | Default |
| ---------------- | ----------------------------------------- | ------- |
| `open`           | `boolean`                                 | —       |
| `defaultOpen`    | `boolean`                                 | `false` |
| `onOpenChange`   | `(open: boolean, details: unknown) => void` | —       |
| `disabled`       | `boolean`                                 | `false` |
| `sx`             | `StyleXStyles`                            | —       |

### CollapsibleTrigger

| Prop       | Type           | Default |
| ---------- | -------------- | ------- |
| `render`   | `ReactElement` | —       |
| `disabled` | `boolean`      | —       |
| `sx`       | `StyleXStyles` | —       |

### CollapsibleContent

| Prop               | Type           | Default |
| ------------------ | -------------- | ------- |
| `keepMounted`      | `boolean`      | `false` |
| `hiddenUntilFound` | `boolean`      | `false` |
| `sx`               | `StyleXStyles` | —       |

All other Base UI Collapsible props are forwarded. Public slots render with
`data-slot="collapsible"`, `data-slot="collapsible-trigger"`, and
`data-slot="collapsible-content"`.
