# Sheet

A Sheet is a slide-in [Dialog](/docs/dialog): the same Base UI dialog behavior —
portal, focus trap, scroll lock, escape-to-close — with the panel pinned to a
screen edge instead of centered. The `side` prop (`top` / `right` / `bottom` /
`left`) drives both positioning and the divider edge. Styled with StyleX design
tokens; never accepts `className`.

## Usage

```tsx
import {
  Sheet,
  SheetClose,
  SheetContent,
  SheetDescription,
  SheetFooter,
  SheetHeader,
  SheetTitle,
  SheetTrigger,
} from "@/components/sheet"
```

```tsx
<Sheet>
  <SheetTrigger render={<Button variant="outline">Open Sheet</Button>} />
  <SheetContent>
    <SheetHeader>
      <SheetTitle>Edit profile</SheetTitle>
      <SheetDescription>
        Make changes to your profile here. Click save when you're done.
      </SheetDescription>
    </SheetHeader>
    <SheetFooter>
      <SheetClose render={<Button variant="outline">Cancel</Button>} />
      <Button>Save changes</Button>
    </SheetFooter>
  </SheetContent>
</Sheet>
```

> **Trigger as a link/other element.** `SheetTrigger` renders a `<button>` by
> default. To style another element as the trigger (e.g. a `Button` or `Link`),
> pass it via the `render` prop — never `className`.

## Sides

`SheetContent` accepts a `side` prop (`right` by default) that pins the panel to
an edge. Left and right sheets span the full height at `w-3/4` (capped at
`max-w-sm`); top and bottom sheets span the full width with auto height.

## Customization

`Sheet` never accepts `className`. Customize it on-system, in order of reach:

1. **`side`** — pick the edge (controls positioning + the divider border).
2. **`sx`** — a typed, per-instance override on any subcomponent that accepts
   `StyleXStyles` and merges last, so it wins. Useful for width overrides on
   `SheetContent` or spacing tweaks on `SheetHeader` / `SheetFooter`.
3. **`createTheme`** — re-skin a whole subtree by overriding token *values*
   (e.g. `colors`, `boxShadow`).

```tsx
import * as stylex from "@stylexjs/stylex"

const styles = stylex.create({
  wide: { maxWidth: "32rem" },
})

<Sheet>
  <SheetTrigger render={<Button variant="outline">Wide</Button>} />
  <SheetContent side="right" sx={styles.wide}>
    <SheetHeader>
      <SheetTitle>Wider sheet</SheetTitle>
      <SheetDescription>
        The default max width is overridden via the last-wins `sx` prop.
      </SheetDescription>
    </SheetHeader>
  </SheetContent>
</Sheet>
```

## API Reference

`Sheet` (aliased from the Base UI dialog `Root`) accepts all [Base UI Dialog Root
props](https://base-ui.com/react/components/dialog) (`open`, `onOpenChange`,
`defaultOpen`, `modal`, etc.) — a sheet is a dialog.

| Subcomponent        | Prop                | Type                                  | Default  |
| ------------------- | ------------------- | ------------------------------------- | -------- |
| `SheetTrigger`      | `render`            | element                               | —        |
|                     | `sx`                | `StyleXStyles`                        | —        |
| `SheetContent`      | `side`              | `"top" \| "right" \| "bottom" \| "left"` | `right` |
|                     | `showCloseButton`   | `boolean`                             | `true`   |
|                     | `sx`                | `StyleXStyles`                        | —        |
| `SheetHeader`       | `sx`                | `StyleXStyles`                        | —        |
| `SheetFooter`       | `sx`                | `StyleXStyles`                        | —        |
| `SheetTitle`        | `sx`                | `StyleXStyles`                        | —        |
| `SheetDescription`  | `sx`                | `StyleXStyles`                        | —        |
| `SheetClose`        | `render`            | element                               | —        |
|                     | `sx`                | `StyleXStyles`                        | —        |
| `SheetOverlay`      | `sx`                | `StyleXStyles`                        | —        |
| `SheetPortal`       | —                   | Base UI Portal                        | —        |

All other props (native attributes, `render`, etc.) are forwarded to the
underlying Base UI dialog primitive.
