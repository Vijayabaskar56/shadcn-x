# Field

A collection of structural components for building form fields. No Base UI
primitive — each subcomponent is a semantic HTML element (or reuses an existing
shadcn-x component like `Label`). Inspired by the Base UI variant's field
conventions.

## Subcomponents

| Component          | HTML Element     | Slot                          |
| ------------------ | ---------------- | ----------------------------- |
| `Field`            | `div`            | `field`                       |
| `FieldSet`         | `fieldset`       | `field-set`                   |
| `FieldLegend`      | `legend`         | `field-legend`                |
| `FieldGroup`       | `div`            | `field-group`                 |
| `FieldContent`     | `div`            | `field-content`               |
| `FieldLabel`       | `label`          | `field-label`                 |
| `FieldTitle`       | `div`            | `field-title`                 |
| `FieldDescription` | `p`              | `field-description`           |
| `FieldSeparator`   | `div`            | `field-separator` / `field-separator-content` |
| `FieldError`       | `div`            | `field-error`                 |

## Usage

```tsx
import { Field, FieldLabel, FieldContent, FieldDescription, FieldError } from "@/components/field"
```

### Error from validation array

```tsx
<Field errors={[{ message: "Required" }, { message: "Too short" }]} />
```

Renders deduplicated error messages. Single errors render inline; multiple errors
render as a list.

## API Reference

### Field

| Prop          | Type                  | Default      |
| ------------- | --------------------- | ------------ |
| `orientation` | `"vertical" \| "horizontal"` | `"vertical"` |
| `sx`          | `StyleXStyles`        | —            |

All forwarded `<div>` props are accepted.

### FieldError

| Prop       | Type                                  | Default |
| ---------- | ------------------------------------- | ------- |
| `children` | `React.ReactNode`                     | —       |
| `errors`   | `Array<{ message?: string } \| undefined>` | —   |
| `sx`       | `StyleXStyles`                        | —       |

If `children` is provided, it renders as-is. Otherwise renders from `errors`.

### Other subcomponents

`FieldLabel`, `FieldSet`, `FieldLegend`, `FieldGroup`, `FieldContent`,
`FieldTitle`, `FieldDescription`, and `FieldSeparator` all accept standard HTML
props and `sx`.
